あなたは熟練のフルスタックエンジニア兼Webデザイナーです。
現在のWebサイトには「お客様の声（Testimonials）」のページ/セクション/カードUIが既に存在します。
今回は新規追加ではなく、既存の「お客様の声」カード（またはデータ）を、下記のSNS自動投稿システム事例に“差し替え”してください。

# 目的
既存Testimonialsの表示構造・デザイン・CSS・コンポーネントを一切崩さず、
表示されるテキスト内容のみをSNS自動投稿システムの事例に差し替える。

# 絶対条件（重要）
- レイアウト/デザイン/コンポーネント構造は維持（UIはそのまま）
- 既存のCSS/クラス名/余白/フォント/色/角丸/影などを変更しない
- 既存カードが複数ある場合：
  - まず「Testimonialsがどこで管理されているか（データ/props/CMS/JSON/markdown等）」を特定
  - 既存カードのうち「先頭の1件」または「指定ID/slugの1件」を本事例に置換
  - 既存の件数・並びは変えない
- 文言は改変しない（改行や句読点の微調整のみOK）
- TypeScript等の型定義があれば整合させ、型エラーを出さない
- 変更差分は最小限。変更した箇所が分かるようコメントを入れる

# 作業手順（やること）
1) コードベース内で testimonial / testimonials / voice / review / customers などで検索して、
   お客様の声の「データ定義」または「カードに流し込むprops」を管理している場所を特定する
   例：/data/testimonials.ts, /constants/testimonials.js, CMSのJSON, markdown など

2) 既存のカード表示ロジックはそのままに、該当データ（1件）を下記内容に差し替える
   - 既存が「title / subtitle / name / company / role / comment」形式なら、それに合わせてマッピングして入れる
   - 既存が「sections（見出し＋本文）」や「bullets（配列）」を持つなら、actions/results を配列で格納する
   - 既存が本文1つの長文フィールドのみなら、下記文章を見出し付きの体裁で1本に整形して入れる
   ※ただし表示枠や項目数は増やさない（UIを増やさない）

3) 実装後、プレビューで既存UIが崩れていないことを確認（レスポンシブ含む）

# 差し替える「お客様の声」コンテンツ（このまま使用）
【事例タイトル】
投稿に追われなくなり、“続く発信”が当たり前になりました

【属性】
業種：サービス業（中小企業）
従業員数：15名／SNS担当：1名（兼務）

【導入前の悩み】
SNSを活用したい気持ちはありましたが、
投稿がどうしても思いつきになり、更新頻度も不安定でした。
複数のSNS（X・Instagramなど）を個別に管理していたため、
「今日は何をどこに投稿したか分からない」「気づいたら数週間止まっていた」
といった状態が続いていました。
担当者も他業務と兼務しており、
忙しくなると真っ先にSNSが後回しになるのが課題でした。

【このSNS自動投稿システムを選んだ理由】
既存の投稿ツールではなく、
自社の発信スタイルや運用体制に合わせて設計してくれる点に魅力を感じました。
「毎日頑張る前提」ではなく、
無理なく続く仕組みを一緒に作るという考え方が、自社に合っていると感じました。

【実施した内容】（箇条書き）
- 現在のSNS運用状況・発信目的の整理
- 投稿頻度・曜日・内容の基本ルール設計
- 複数SNS（X・Instagramなど）への予約・自動投稿設定
- 投稿管理を一元化できるシンプルな管理画面の構築
- 運用開始後の改善（投稿内容・タイミングの調整）
「頑張って続ける」のではなく、続いている状態を仕組みで作る設計をしてもらいました。

【得られた成果・変化】（箇条書き）
- 投稿が止まることがなくなった
- SNS更新にかかる心理的・時間的負担が大幅に軽減
- 投稿内容やトーンが安定し、ブランド感が整った
- 「ちゃんと発信している会社」という印象を持たれるようになった
特に大きかったのは、SNSが“気合いでやるもの”から“自然に回るもの”に変わったことです。

【これから検討する方へ】
SNSは続けることが一番難しいと思っていましたが、
仕組みを整えるだけで、ここまで楽になるとは思いませんでした。
発信を止めずに、無理なく続けたい企業やチームには、
とても相性の良いシステムだと思います。

# 形式の注意（既存カード項目に合わせる）
- 既存が短文コメント＋氏名/会社名のみ必須なら：
  - title（見出し）＝上記タイトル
  - comment（本文）＝「導入前の悩み」〜「これから検討する方へ」を1本にまとめて入れる
  - name/company/role が必須なら、架空で自然な表記にする（例：株式会社◯◯ 広報担当 / ◯◯様）
  ※UIの項目数や表示枠は増やさない

# 納品要件（必ず出す）
- 変更したファイル名とパス
- どのデータ（何件目/どのID）を差し替えたか
- 既存UIが崩れていない確認結果（簡単でOK）
- 可能なら最小差分になるように変更し、差分が追いやすい形にする
